# 更新日志 (CHANGELOG)

- **v1.4.14**:
  - 全量代码自查与鲁棒性增强：
    - 将网络请求超时从 10s 提升至 20s，降低网络波动导致的抓取失败率。
    - 优化内部日志逻辑，确保生产环境下运行稳定。
- **v1.4.13**:
  - 优化启动推送体验：引入“会话静默初始化”机制。每次插件启动或重载后的第一次检查仅同步各账号的最新 ID，不再推送离线期间积累的历史动态，避免“启动即炸群”。
- **v1.4.12**:
  - 增强置顶微博过滤逻辑，支持多种置顶标识识别（isTop, is_top, title等），解决部分账号一直获取到置顶微博的问题。
  - 优化新微博推送逻辑，支持一次性推送多个最新动态（按时间顺序从旧到新），不再只推送最新一条。
- **v1.4.11**:
  - 修复热重载后后台监控任务无法自动启动的问题。
  - 增加数据目录自动迁移逻辑，兼容旧路径数据，防止升级后首轮推送跳过。
- **v1.4.10**:
  - 代码深度优化与自查修复：
    - 统一使用 `pathlib.Path` 方法处理数据加载与保存。
    - 增强后台监控任务的异常捕获，单个账号失败不再影响其他账号。
    - 优化 `on_loaded` 钩子，防止重复创建监控任务。
- **v1.4.9**:
  - 遵循 AstrBot 插件开发最佳实践进行优化：
    - 使用 `StarTools.get_data_dir()` 规范化数据存储路径。
    - 将后台监控任务的启动移至 `@filter.on_astrbot_loaded()` 钩子中。
    - 增强 `clean_text` 方法的类型安全性。
- **v1.4.8**:
  - 修正 `metadata.yaml` 中的 `name`, `desc` 和 `repo` 字段以符合验证要求。
- **v1.4.7**:
  - 配置 `repo_url` 以支持 AstrBot 强制从仓库地址更新的功能。
- **v1.4.6**:
  - 添加文档按钮链接。
- **v1.4.2**:
  - 修复 `/weibo_verify` 指令在某些情况下可能出现的 `KeyError: 'user'` 崩溃问题。
  - 增强 Cookie 验证逻辑的健壮性。
- **v1.4.1**:
  - 新增 `request_interval` 配置项，支持设置多个账号抓取之间的延迟时间（默认 5s），提高稳定性。
- **v1.4.0**:
  - 支持向多个推送目标会话推送动态。
  - 优化 `target_conversation_id` 配置项为列表类型。
- **v1.3.1**: 
  - 修复自定义消息格式不支持换行的问题（现在可以使用 `\n` 进行换行）。
- **v1.3.0**: 
  - 新增 `/weibo_verify` 指令，验证 Cookie 是否有效。
  - 支持自定义消息格式，提供 `{name}`, `{weibo}`, `{link}` 变量。
  - 优化微博删除后的逻辑，采用 ID 数值比较防止重复推送。
- **v1.2.0**: 
  - 新增 `/weibo_check` 指令，支持立即执行监控检查并发送最新微博（用于测试）。
- **v1.1.2**: 
  - 修复 `WeiboMonitor object has no attribute 'config'` 运行时错误。
- **v1.1.1**: 
  - 修复 WebUI 设置页面显示为空的问题。
  - 优化配置项定义，支持列表类型。
- **v1.1.0**: 
  - 增加 Cookie 配置项以增强稳定性。
  - 优化请求头，模拟移动端访问。
  - 增加 `/get_umo` 指令方便获取推送目标。
