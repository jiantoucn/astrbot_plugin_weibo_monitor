# 更新日志 (CHANGELOG)

- **v1.11.3**:
  - **修复**: 修复 `cookie_notification_target` 配置解析的 bug，确保即使在配置值为列表或包含空格等情况下，也能正确解析并向指定目标发送通知。

- **v1.11.2**:
  - **新功能**: 增加 `cookie_notification_target` 配置项。允许用户指定 Cookie 失效时的通知目标，支持将通知发送到特定的账号或群聊，而不必发送到所有监控推送目标。
  - **优化**: 完善 Cookie 失效提醒机制。未配置 `cookie_notification_target` 时保持原样（向所有目标推送）；配置后则仅向指定目标推送，保护隐私并减少干扰。

- **v1.11.1**:
  - **新功能**: 增加 Cookie 健康检查功能。在每次抓取内容前检测 Cookie 是否有效，若失效则向用户发送一次提醒消息，避免因 Cookie 失效导致静默抓取失败。
  - **优化**: 完善 Cookie 失效提醒逻辑，同一失效周期内仅提醒一次，避免刷屏。

- **v1.11.0**:
  - **新功能**: 初始化监控时，自动将获取到的历史微博记录到每日日志（需开启 `enable_daily_log`），避免历史微博丢失。
  - **新功能**: 微博发布时间现在会从微博数据中解析并精准保存到日志中，不再统一使用记录时间。
  - **优化**: 完善微博时间解析逻辑，支持“刚刚”、“x分钟前”、“x小时前”、“昨天”、“MM-DD”及标准格式等多种微博时间显示方式。
  - **优化**: 增强日志记录去重逻辑，防止初始化或重复检查时在日志中产生重复记录。

- **v1.10.3**:
  - **新功能**: 增加 `/weibo_status` 命令，查看当前监控状态（监控账号数、推送目标、检查间隔等）。
  - **新功能**: 增加 `/weibo_summary` 命令，手动触发昨日总结推送。
  - **优化**: 手动触发获取的微博不计入每日推送日志。

- **v1.10.2**:
  - **修复**: 修复每日总结在插件重载时重复发送的问题，现在只有到达预定时间才会发送（首次更新插件依然会推送）。

- **v1.10.1**:
  - **优化**: 增强日志系统，解决 setup_logging 重复调用导致的 handler 重复添加问题。
  - **优化**: 增强 HTTP 客户端健壮性，添加连接池限制和请求信号量，防止高并发导致的问题。
  - **优化**: 增强主循环稳定性，添加指数退避策略，当连续错误发生时自动延长等待时间，避免频繁重试导致资源耗尽。
  - **修复**: 修复每日总结跨日边界问题，确保第二天能够正常触发推送。

- **v1.10.0**:
  - **新功能**: 增加运行日志功能 (plugin.log)，支持滚动记录和自定义文件大小，方便排查问题。
  - **新功能**: 增加每日推送记录功能，以日期命名 (如 20260316.log)，记录微博名、内容和链接，采用 JSON 格式，方便后续统计和分析。
  - **新功能**: 增加每日总结推送功能，每天 08:00 (可配置) 自动统计并推送前一日监控到的微博动态数量。
  - **优化**: 全面升级日志系统，提供更详尽的运行状态记录和错误堆栈追踪，方便定位复杂问题。

- **v1.9.0**:
  - **新功能**: 增加关键词白名单功能（whitelist_keywords），只有微博正文包含白名单关键词时才推送。（[#3](https://github.com/jiantoucn/astrbot_plugin_weibo_monitor/issues/3)）
  - **优化**: 完善过滤规则逻辑，先检查屏蔽词（黑名单），后检查白名单，确保用户体验流畅。

- **v1.8.1**:
  - **优化**: 允许插件在未配置 Cookie 时启动，但会跳过监控循环并输出警告日志，方便用户进入后台进行配置。

- **v1.8.0**:
  - **重大变更**: Cookie 改为必填项，未配置 Cookie 时插件将无法启动，确保数据抓取稳定性。
  - **启动检查增强**: 插件初始化时自动验证 Cookie 配置，未配置时抛出明确错误提示。
  - **文档全面更新**: 完善 README.md 文档，补充所有配置项说明、Cookie 获取教程和常见问题解答。
  - **配置说明优化**: 详细说明 `weibo_urls` 支持的三种格式（UID、个人主页URL、用户名URL）。
  - **用户体验提升**: 新增常见问题 FAQ 部分，解答用户使用过程中可能遇到的问题。

- **v1.7.7**:
  - **稳定性大幅提升**: 对全代码进行深度审查和优化，确保在极端网络环境、数据损坏等场景下都能稳定运行。
  - **HTTP请求优化**: 为HTTP客户端添加自动重试机制（最多重试2次），提升网络不稳定时的请求成功率。
  - **数据持久化安全增强**: 采用原子写入机制保存数据，避免程序意外中断导致数据文件损坏；增加数据损坏自动备份功能，防止数据丢失。
  - **错误处理增强**: 对JSON解析、数据结构校验、URL解析等关键路径添加更完善的异常捕获和处理逻辑。
  - **代码健壮性优化**: 增加对各种异常数据结构的校验，避免因接口返回格式变化导致程序崩溃。

- **v1.7.6**:
  - **核心逻辑优化**: 精细化微博监控初始化逻辑，能更准确地在日志中区分“首次安装初始化”与“日常启动会话同步”。
  - **代码健壮性增强**: 优化了随机抖动 (Jitter) 的计算逻辑，确保在极端配置下仍能稳定运行，并对关键监控周期进行了深度自审。
  - **日志信息完善**: 改进了监控周期内的调试日志，方便用户更直观地了解下一次检查的预计时间。

- **v1.7.5**:
  - **随机间隔功能**: 为“检查间隔”和“账号请求间隔”引入了随机浮动参数，模拟真人无规律操作，提高安全性。
  - **新配置项**:
    - `check_interval_jitter`: 检查间隔随机浮动（分钟）。
    - `request_interval_jitter`: 请求间隔随机浮动（秒）。

- **v1.7.4**:
  - **新功能**: 增加区分原创和转发微博的功能。
  - **配置项**: 
    - `send_original`: 是否推送原创微博 (默认开启)。
    - `send_forward`: 是否推送转发微博 (默认开启)。

  - **指令调整**:

    - `/weibo_check_all`: 新增指令，保留原有功能，抓取列表中所有账号的最新动态。

- **v1.7.1**:
